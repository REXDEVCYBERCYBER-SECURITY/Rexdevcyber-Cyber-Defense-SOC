# REXDEVCYBER: ศูนย์ปฏิบัติการความปลอดภัยไซเบอร์ยุคใหม่

การออกแบบ UI เน้น **โทนสีมืดคอนทราสต์สูง** สไตล์ไซเบอร์พังค์ โดยใช้สี Teal (เขียวอมฟ้า) และสีแดงนีออนบนพื้นหลังมืด เพื่อเน้นความเร่งด่วนของภัยคุกคามและความปลอดภัยขั้นสูง. แนวทางนี้สอดคล้องกับการออกแบบไซเบอร์พังค์ที่มักใช้ไฟนีออนและพื้นหลังสีเข้ม รวมถึงการใช้ดีไซน์แบบ **glassmorphism** (แผงโปร่งใสเบลอพื้นหลังเล็กน้อย) เพื่อสร้างความลึกให้กับหน้าจอ. บริบทของกริดและแอนิเมชันเล็กน้อย (เช่น การกระพริบเตือนหรือเส้นเลเซอร์ที่ไหลตามข้อมูล) ช่วยสื่อถึงข้อมูลการรันไทม์และระบบโลกไซเบอร์ระดับสูง

- **พาเล็ตสี Teal & แดงนีออน**: โทนสีหลักคือ Teal กับแดงสว่างบนพื้นหลังดำ ให้คอนทราสต์สูง ช่วยให้ข้อความเตือนและสถานะความเสี่ยงโดดเด่น  
- **เลย์เอาต์ Glassmorphic**: ใช้แผงใสโปร่งใสแบบกระจก (frosted glass) ซ้อนกับพื้นหลังดาร์กเพื่อเน้นความสะอาดและความล้ำยุค  
- **เวกเตอร์ไดนามิก**: มีกราฟิกแผนที่และปุ่มจุดเคลื่อนไหวตามข้อมูล เช่น แกนพิกัดโลกหรือเลเซอร์โค้ดที่สะท้อนเชื่อมต่อเครือข่าย  
- **แอนิเมชันเล็กน้อย**: ใช้แอนิเมชันจุดประกายไฟหรือการกระพริบของ UI เพื่อให้เกิดฟีดแบคแบบเรียลไทม์ (เช่น สถานะเตือน)  

## ฟีเจอร์โมดูลหลัก

### 1. 🛡️ คอนโซลเทอร์มินัล SOC (`Terminal.tsx`)
- **Auto-Suggest คำสั่ง**: มีเมนูแนะนำคำสั่งและพารามิเตอร์อัตโนมัติขณะพิมพ์ รองรับการเลื่อนเลือกด้วยปุ่มลูกศร ↑↓ และกด Tab เพื่อเติมคำสั่งให้เร็วขึ้น (คล้ายกับส่วนขยายแบบ “autocomplete”).  
- **ระบบเทอร์มินัลจำลอง**: ใช้คอมโพเนนต์ React เช่น *react-console-emulator* หรือ *react-terminal-component* ที่สามารถ “จำลองเชลล์ยูนิกซ์ในเบราว์เซอร์” ได้อย่างละเอียด. Terminal นี้สามารถเก็บประวัติคำสั่งและรันผลลัพธ์แบบอะซิงโครนัสหลังจากส่งคำสั่งแล้ว.  
- **คำสั่งเฉพาะด้านความมั่นคง**: สร้างชุดคำสั่งพิเศษ เช่น `help` (แสดงรายการคำสั่งที่มี), `status` (รายงานสถานะระบบและไฟร์วอลล์), `scan <domain>` (สร้างรายงานช่องโหว่จำลองของโฮสต์หรือโดเมน), `ai <query>` (เชื่อมต่อกับ AI ผู้ช่วย), `decrypt` (จำลองการตรวจสอบถอดรหัสอัตโนมัติ), `clear` (ล้างคอนโซล), `exit` (จบเซสชัน) เป็นต้น. แต่ละคำสั่งนี้เชื่อมต่อกับฟังก์ชันในโค้ดหลักเพื่อคำนวณข้อมูลความมั่นคงและแสดงบนหน้าต่างคอนโซล  

### 2. 📁 โฟลเดอร์รักษาความปลอดภัย (DriveVault.tsx & `drive.ts`)
- **Firebase OAuth & Google Drive**: ใช้ Firebase Authentication ด้วย Google Sign-In เพื่อให้ผู้ใช้ล็อกอินผ่านบัญชี Google ของตน หลังจากล็อกอินสำเร็จจะได้ *Google Access Token* สำหรับเรียกใช้ Google Drive API.  
- **การอัปโหลดไฟล์ (Dual-Phase Upload)**: อัปโหลดข้อมูลรายงาน (รูปภาพ, ไบนารี, หรือไฟล์ .txt) ไปยัง Google Drive โดยใช้วิธี multipart upload เช่น `uploadType=resumable` ตามข้อแนะนำของเอกสาร Drive API. ระบบจะจัดการกำหนด `Content-Type` ให้ตรงกับไฟล์และสนับสนุนการส่งข้อมูลเป็นขั้นตอน (chunked) เพื่อให้สามารถแสดงความคืบหน้าการอัปโหลดได้ (ตามเอกสารของ Google Drive API).  
- **ตัวจัดการไฟล์**: มีอินเทอร์เฟซสำรวจไฟล์บน Drive แสดงรายการไฟล์ในโฟลเดอร์ ผู้ใช้สามารถค้นหา กรอง ดาวน์โหลด หรือลบไฟล์ได้จากหน้าเว็บโดยตรง. นอกจากนี้ยังสามารถสร้างไฟล์ข้อความ (ไฟล์ .txt) เพื่อบันทึกภัยคุกคามไว้ใน Drive ได้ทันที (สร้าง Threat Intelligence Dossier).  
- **การตรวจสอบความสมบูรณ์**: ฟีเจอร์สแกนไฟล์จะคำนวณค่า SHA-256 ของไฟล์และวิเคราะห์ค่า Entropy เพื่อตรวจจับลักษณะไฟล์ที่อาจบ่งชี้ถึงภัยคุกคามหรือมัลแวร์ (มองภาพเป็นสแกนน้ำหนักข้อมูลและรูปแบบบิตของไฟล์)  
- **ขอบเขต OAuth**: ก่อนล็อกอินด้วย Google Auth จะมีการเรียก `provider.addScope('https://www.googleapis.com/auth/drive')` เพื่อขอสิทธิ์เข้าถึง Google Drive. สิทธิ์นี้ต้องลงทะเบียนใน Google Cloud Console โดยกำหนดในหน้าจอการขอสิทธิ์ OAuth Consent Screen ด้วย.  

### 3. 🌐 วิดเจ็ตเฝ้าระวังภัยคุกคาม (ThreatMap.tsx)
- **แผนที่ภัยคุกคามเรียลไทม์**: แสดงข้อมูลการโจมตีไซเบอร์ทั่วโลกผ่านกราฟิกแผนที่แบบไลฟ์ จุดหรือดาร์ทบนแผนที่จะแสดงตำแหน่งต้นทางและปลายทางของการโจมตี พร้อมเส้นนำทาง. แต่ละจุดใช้ **สี** ระบุระดับความรุนแรงของภัยคุกคาม (โดยทั่วไปสีแดงแสดงระดับอันตรายสูง).  
- **ดัชนีภัยคุกคาม (Threat Level)**: ให้ภาพรวมค่าความเสี่ยงในแบบกราฟหรือเกจ เช่น กราฟแท่งสีที่ปรับเปลี่ยนตามจำนวนเหตุการณ์สำคัญล่าสุด เพื่อช่วยวัดระดับภัยคุกคามในปัจจุบัน.  
- **การแจ้งเตือน Honeypot**: แสดงข้อความแจ้งเตือนจากตัวหลอกล่อ (honeypot) ที่ติดตั้งไว้กับระบบ ซึ่งใช้สำหรับซักถามหรือดักจับกิจกรรมที่ผิดปกติ (สร้างเป็นการจำลองเหตุการณ์แจ้งเตือนภัย).  

## การตั้งค่าและดีพลอย

### 1. ตัวแปรสภาพแวดล้อม (Environment Variables)
กำหนดค่าความลับต่าง ๆ ในไฟล์ `.env` ตามตัวอย่าง `.env.example` ในโปรเจค เช่น
```env
GEMINI_API_KEY=YOUR_GEMINI_KEY
```
(`GEMINI_API_KEY` สำหรับเรียกใช้บริการ AI ของ Google เป็นต้น) รวมถึงค่าการตั้งค่าอื่น ๆ เช่น URL ของ Firebase ถ้ามี

### 2. การกำหนดค่า Firebase & Google Drive
- **Google Cloud Console**: สร้างโปรเจคใหม่และเปิดใช้งาน **Google Drive API**. ตั้งค่าหน้าจอ **OAuth consent screen** โดยระบุข้อมูลแอปและสิทธิ์ที่ร้องขอ (เช่น `https://www.googleapis.com/auth/drive`). สร้าง OAuth 2.0 Client ID ประเภท **Web application** แล้วลงทะเบียน Authorized JavaScript origins (เช่น `http://localhost:3000`) และ Authorized redirect URIs สำหรับโหมด redirect.  
- **Firebase Console**: ในโปรเจค Firebase เปิดใช้งาน Google Authentication (Google Sign-In) และเพิ่ม Authorized domain (เช่น `localhost`). ดาวน์โหลดไฟล์ `firebase-applet-config.json` จากการตั้งค่าโปรเจค Firebase แล้วนำไปวางในโฟลเดอร์โปรเจค (src) เพื่อเชื่อมต่อกับคลาวด์.  
- **ไฟร์วอลล์และความปลอดภัย**: ไม่เปิดเผย Client Secret ทางฝั่งไคลเอ็นต์ กำหนด Credential (Client ID และ API Key) ให้ล็อกกับโดเมนหรือแอปที่กำหนดเท่านั้น.

### 3. คำสั่ง Quickstart
ติดตั้งและรันเซิร์ฟเวอร์พัฒนา ใช้คำสั่ง npm ดังนี้:
```bash
npm install   # ติดตั้งไลบรารีที่จำเป็น
npm run dev   # สตาร์ทเซิร์ฟเวอร์โหมด development (host: 0.0.0.0, port: 3000)
npm run build # คอมไพล์โปรเจคสำหรับ production
```

## สถาปัตยกรรมโปรเจค
โครงสร้างไฟล์หลักของโปรเจคประกอบด้วย:  
- `src/App.tsx` – กำหนดเลย์เอาต์หลักของแอป (routing, ส่วนหัว, footer) และจัดการแท็บเลือกโมดูลต่าง ๆ.  
- `src/components/Header.tsx` – แถบเมนูบนสุด พร้อมนาฬิกา SOC แบบเรียลไทม์และข้อมูลล็อกอิน.  
- `src/components/ParticlesBg.tsx` – เอฟเฟ็กต์อนุภาคไดนามิกพื้นหลัง (Canvas particle flow).  
- `src/components/ThreatMap.tsx` – คอมโพเนนต์แสดงแผนที่ภัยคุกคามทั่วโลก (SVG/Canvas) ที่อัพเดตแบบเรียลไทม์.  
- `src/components/Terminal.tsx` – คอมโพเนนต์คอนโซลเทอร์มินัลอินเทอร์แอคทีฟ พร้อมระบบเติมคำสั่งและแสดงผลเหมือน CLI.  
- `src/components/DriveVault.tsx` – UI สำหรับจัดการไฟล์ใน Google Drive (สืบค้น, อัปโหลด, ดาวน์โหลด, ลบ, สแกนความสมบูรณ์).  
- `src/lib/drive.ts` – ไลบรารีจับการเรียกใช้ Firebase Auth และ Google Drive API client (โค้ด backend สำหรับฟังก์ชันจัดการไฟล์).  
- ไฟล์อื่น ๆ ได้แก่ `index.css` (ส่วนกลางสไตล์ Tailwind), `types.ts` (interface TypeScript สำหรับ telemetry), และไฟล์ config (`firebase-applet-config.json`, `tsconfig.json` เป็นต้น).

## บทสรุป
โครงการ **REXDEVCYBER** รวบรวมแนวทางดีไซน์และเทคโนโลยีสมัยใหม่สำหรับ SOC Dashboard แบบเต็มรูปแบบ โดยมีการผสาน UI สไตล์ไซเบอร์พังค์ (สีสันนีออน โทนมืด เอฟเฟกต์กระจกโปร่งใส) เข้ากับฟีเจอร์การใช้งานจริง เช่น คอนโซลคำสั่ง, แผนที่ภัยคุกคาม, และระบบจัดการไฟล์บนคลาวด์. ในเชิงเทคนิค ใช้ Firebase Authentication (Google Sign-In) เพื่อยืนยันตัวผู้ใช้และเรียกใช้ Google Drive API โดยได้รับ *access token* หลังล็อกอิน. นอกจากนี้ ผู้พัฒนาต้องเปิดใช้งาน Google Drive API บน Google Cloud Console และกำหนด OAuth Consent Screen ให้เรียบร้อย. ส่วนคอมโพเนนต์ที่ใช้สามารถเลือกจากไลบรารีโอเพ่นซอร์ส เช่น *react-console-emulator* หรือ *react-terminal-component* เพื่อทำให้เทอร์มินัลเว็บตอบสนองเหมือนเชลล์จริง. ทั้งหมดนี้ผนวกเข้าด้วยกันในสถาปัตยกรรมโปรเจคที่ชัดเจน เพื่องานดีพลอยที่ราบรื่นและประสบการณ์การใช้งานผู้ใช้ที่ทันสมัย.

**อ้างอิง:** แนวทางออกแบบไซเบอร์พังค์, เอกสาร Google Drive API และ OAuth, เอกสาร Firebase Authentication, ตัวอย่างคอมโพเนนต์คอนโซล React, และแนวปฏิบัติด้านความปลอดภัย OAuth.
